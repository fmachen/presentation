Les libs/frameworks front

MVVM

  • Model : gérer les données (mémoriser, conserver, restituer). Pas de traitement.
  • Vue : l'interface utilisateur. Elle est interactive -> Si l'utilisateur fait une action, la vue doit s'adapter en conséquence.

VueModèle

  • Lien entre Model et Vue.
  • La liaison à double sens :
    • un changement dans le modèle doit être actualisé dans la vue (data-binding)
    • une action dans la vue doit être transmise au modèle

Vue is just js


<html>

directive v-bind


<div id="vm5">>
    <span class="label" v-bind:class="type" 
        v-on:mouseover="changeType">{{type}}</span>
    <span class="label" :class="type" 
        v-on:mouseover="changeType">{{type}}</span>
</div>
    

new Vue({
    el: '#vm5',
    data: {
        type: 'label-primary'
    },
    methods: {
        changeType: function() {
            this.type = (this.type == 'label-primary') ?
                 'label-success' : 'label-primary';
        }
    }
});
{{type}}
