Les libs/frameworks front

MVVM

  • Model : gérer les données (mémoriser, conserver, restituer). Pas de traitement.
  • Vue : l'interface utilisateur. Elle est interactive -> Si l'utilisateur fait une action, la vue doit s'adapter en conséquence.

VueModèle

  • Lien entre Model et Vue.
  • La liaison à double sens :
    • un changement dans le modèle doit être actualisé dans la vue (data-binding)
    • une action dans la vue doit être transmise au modèle

Les utilitaires

Raccourcis pour les directives

  • v-on:click devient @click
  • v-bind:class devient :class
  • ... v-show, v-if, v-else, v-for ...

Raccourcis pour les événements

  • On peut ajouter aux événements des modificateurs
    • .stop : stoppe la propagation
    • .prevent : empêche l'action implicite
    • .capture : utilise le mode capture
    • .self : la cible doit être l'élément lui-même
    • .once : déclenche une seule fois

var vm11 = new Vue({
    el: '#vm11',
    data: {
        person: {lastname: "Foo", firstname: "Bar"}
    },
    computed: {
        fullname: {
            get: function () {
                return this.person.lastname 
                    + ' ' + this.person.firstname; 
            },
            set: function (value) {
                var nom = value.split(' ');
                this.person.lastname = nom[0];
                this.person.firstname = nom[1];
            }
        }
        }
    });
